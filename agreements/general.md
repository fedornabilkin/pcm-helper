# Общие соглашения проекта

## Назначение

Этот документ фиксирует базовые договоренности для разработки PCM Helper. Новые частные соглашения следует добавлять отдельными `.md` файлами в директорию `agreements`.

## Структура

- Основное приложение находится в `app`.
- Исходный код приложения хранится в `app/src`.
- Домен PCM хранится в `app/src/pcm`.
- Домен networker хранится в `app/src/networker`.
- Общий переиспользуемый код хранится в `app/src/core`.
- Планы работ хранятся в директории `plans` в формате `.md`.

## Технологии

- Базовый стек: Vue 3, Vite, Vue Router, TypeScript, Bulma, D3.
- Новая архитектура и новые Vue-компоненты должны соответствовать правилам из `agreements/architecture.md`.
- Не смешивать JS и TS в рамках одной новой функциональности без причины.
- Импорты через алиас `@` использовать для переходов от `src`, относительные импорты оставлять для соседних файлов одного модуля.

## Данные и состояние

- JSON-данные считать частью доменной модели: изменения структуры данных должны сопровождаться обновлением entity/builder/service слоя.
- Данные из `localStorage` всегда читать через безопасный парсер с fallback-значением.
- DTO для импорта/экспорта должны иметь версионирование схемы после первого несовместимого изменения.
- Сохранение пользовательских данных не должно зависеть от демо-данных по умолчанию.

## Качество кода

- Перед завершением значимых изменений запускать `npm run build` из директории `app`.
- Скрипт тестов должен указывать на реально используемый test runner.
- `@ts-ignore`, `any` и неинициализированные поля допустимы только как временные решения и должны попадать в план устранения.
- Публичные методы сервисов должны возвращать один ожидаемый тип либо явно отражать отсутствие результата через `undefined`/`null`.
- Ошибки пользовательского ввода и импорта файлов должны обрабатываться без падения приложения.

## Интерфейс

- Сохранять существующий визуальный стек на Bulma, пока не принято отдельное решение о миграции.
- Приоритетно использовать инструменты Bulma: готовые компоненты, сетку, helpers и utility-классы.
- Кастомный CSS добавлять только когда средствами Bulma нельзя получить нужное поведение или вид без ухудшения читаемости.
- Общие цвета и переопределения Bulma держать в `app/src/assets/bulma/root-custom.css`.
- Компоненты страниц размещать в `app/src/components/page`, переиспользуемые доменные компоненты - внутри соответствующего домена.
- Тексты интерфейса должны храниться в корректной UTF-8 кодировке.

## PCM-цвета

- Проект основан на PCM, поэтому цвета являются частью доменной семантики.
- На сайте допустимы шесть PCM-гамм: синяя, фиолетовая, оранжевая, коричневая, желтая и красная.
- Маппинг типов и цветов должен быть неизменным:
  - логик - синий;
  - упорный - фиолетовый;
  - душевный - оранжевый;
  - мечтатель - коричневый;
  - бунтарь - желтый;
  - деятель - красный.
- Нельзя использовать цвет PCM-типа для другого типа или состояния.
- Новые UI-элементы, связанные с PCM-типами, должны брать цвет из этого маппинга, а не задавать произвольное значение.
- Если нужен оттенок для hover, border, background или disabled-состояния, он должен оставаться внутри гаммы соответствующего PCM-типа.

## Документация

- Архитектурные правила проекта описаны в `agreements/architecture.md`.
- Каждый план в `plans` должен соответствовать правилам из `agreements/plans.md`.
- Завершенные планы не удалять: переносить статус в документе или создавать follow-up план.
- Технические решения, влияющие на архитектуру, фиксировать отдельным соглашением в `agreements`.
