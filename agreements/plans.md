# Соглашение по формированию планов

## Назначение

Это соглашение задает единый формат планов работ в директории `plans`.

## Имена файлов

- Каждый файл плана должен быть в формате `.md`.
- В названии файла обязательно указывать дату создания плана в формате `YYYY-MM-DD`.
- Рекомендуемый формат имени: `YYYY-MM-DD-short-plan-name.md`.
- Название после даты должно быть коротким, на английском, в kebab-case.

## Структура плана

- План должен содержать цель, контекст, группы задач и критерии готовности.
- Задачи плана группировать по смысловым группам.
- Каждая группа должна иметь буквенный идентификатор и название.
- Формат заголовка группы: `## Группа A: Название группы`.
- Задачи внутри группы именовать буквой группы и номером: `A1`, `A2`, `B1`, `B2`.
- Выполненные задачи отмечать Markdown-чекбоксом.

## Требования к задачам

- Каждый пункт плана должен быть атомарным.
- Каждый пункт плана должен быть независимым настолько, насколько это возможно.
- Задача должна описывать один проверяемый результат.
- Задачи нужно формировать так, чтобы работу можно было прервать, переключиться на другие задачи и продолжить выполнение плана в другое время.
- В тексте задачи должен быть понятен следующий практический шаг без обращения к устному контексту.
- Если задача зависит от другой, зависимость нужно указать явно в тексте задачи.

## Формат задач

Использовать такой формат:

```md
- [ ] A1. Кратко описать действие и ожидаемый результат.
- [x] A2. Отметить выполненную задачу чекбоксом.
```

## Обновление планов

- При выполнении задачи менять `[ ]` на `[x]`.
- Не удалять выполненные задачи из активного плана.
- Если задача стала неактуальной, оставлять ее в плане и добавлять краткое пояснение.
- Если во время работы появляется новая задача, добавлять ее в подходящую группу с новым номером.
- Если новая задача не относится к существующим группам, создавать новую группу со следующей буквой.
